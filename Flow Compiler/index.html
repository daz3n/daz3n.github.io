Flow Compiler

Overview

  • What is it?
  • Why was it made?
  • Who is it for?
  • What can it do?

What is it?

Flow compiler is a tiny bytecode compiler made to fit into a game editor. In order to stay small, it supports a very limited set of keywords, and enforces a C-like syntax to allow for most expressions to be evaluated in a single pass.

Why was it made?

Mainly to gain a deeper understanding of compilers, and to see if I could. Until the language has some capabilities for linking to external code, the language will remain very restricted in its capabilities.

Who is it for?

After I’m more confident in its’ robustness and safety, everyone! Hopefully the compiler can act as a foundation for other, cooler projects that other people make.

What can it do?

Currently, the compiler can compile almost any C source file correctly into bytecode, as long as the file doesn’t call into any external functions or libraries, including syscalls. This severely limits the usage of the language, so one of the higher priorities for the language is getting this in.
