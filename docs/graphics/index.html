Graphics

Overview

The visual component of a game is often the most heavily praised - or criticised - before anything else. Because it plays such an important role in how the final product is recieved, we’ve spent a long time on improving both the graphics workflow and the speed of the scene draw inside the engine.

How To

Brief

  • Create an entity
  • Add a Render Component
  • Set the model, as well as the material to render it with. textures are applied on a per-material basis.

Detailed

After importing assets into the editor, the next question might be ‘well, I can’t see them in the scene, so where are they?’. Not to worry young padawan, for they are floating, but not lost, in the ether of asset management. And if you haven’t imported any assets yet, not to worry - the editor comes with some default models that you can use to get a feel for how your scene should look early on.

Create an Entity

First, you need to create an entity in the scene. You should be able to right click on either the viewport window or the scene hierarchy panel to bring up a context menu. Select ‘empty entity’ - you should find that the entity gets automatically selected, and appears in the entity inspector window.

Add a RenderComponent

At this point, the entity is still invisible. If you deselect the entity, the only way to select it again is through clicking on it in the scene hierarchy panel. In the inspector, there is a search bar, where you can search for components and scripts to attach to your entity. Type in RenderComponent and select the componenet that comes up to add it to the entity. Now, you should (hopefully) see a white cube pop into existence in the scene. The inspector panel should also show the newly added render component that’s been attached to the entiy.

From this point you can change the settings on the component, so get creative! Change the mesh, or the material information to see what sorts of results you can get from it. Or author your own model in an external tool, and just use that instead!
