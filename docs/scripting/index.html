Scripting

Scripting is provided in the engine through a custom scripting language - Flow. Although the dialect is slightly unique, it is heavily inspired by C.

Primitives

type             width     info
i8 1 an integer value from -128 to 127
u8 1 an integer value from 0 to 255
char 1 any ascii character
bool 1 a value representing either true or false
i16 2 an integer value from -32768 to 32767
u16 2 an integer value from 0 - 65535
wchar 2 any unicode character
i32 4 an integer value from -2147483648 to 2147483647
u32 4 an integer value from 0 to 4294967295
i64 8 an integer value from -9223372036854775808 to 9223372036854775807
u64 8 an integer value from 0 to 18446744073709551615

Syntax

Declaring a variable

	i8 x = 0;
	i8 y{0};
	i8 z;

Declaring an array

	i8 array1[5];
	i8 array2[5 * 2 + 1];

Trying to create an array of size 0 or negative size will result in a compile-time error. Arrays can also be procedurally created at compile-time by accessing compiler metadata.

Defining a class

	class myClass
	{
		...
	}

A class can have an optional constructor and destructor, although they aren’t compulsory. If one isn’t provided, then the compiler will attempt to create a default constructor.

Defining a function

	i8 add(i8 a, i8 b)
	{
		return a + b;
	}

note that default arguments aren’t currently supported in flow. Here, a temporary of type i8 is returned from the function ‘add’.

Defining an enumeration

	enum myEnum
	{
		value1,
		value2,
		value3
	}

Control Flow Statements

	if x > 500 { ... } 
	else if x > 200 { ... } 
	else { ... }

	while x > 5 { ... }

Just like in many other languages. Any boolean expression after the if keyword is evaluated at runtime to pick a branch.

Notes

During compilation, checks can be made to require arrays of a minimum size to be passed into functions. In other words, syntax like this is legal.

void my_func(int my_input_array[5])
{
	...
}

Here, my_func is able to take any array whose size is known at compile time, as long as its’ size is equal to or greater than 5. The array isn’t copied, instead a reference to the beginning of the array is passed into the function in a similar style to C++. The main difference is that this allows an additional layer of compile-time security - trying to use raw pointers or arrays that are too small will result in a much safer, and more predictable, compile time error.
