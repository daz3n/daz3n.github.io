Assets

Overview

Assets are a foundational building block of every game. Assets are permanent files that are read by the engine in order to provide the gameplay experience. Some can be created procedurally or programatically, using tools that the engine provides, whilst others need to be imported from external tools (like blender or houdini).

In order for the editor to function, additional metadata is attached to each asset when it’s imported in. This means that once an asset is imported into the engine, it can only be moved, renamed or deleted. If it’s overwritten by another file outside of the editor environment, then the engine could break.

Animations

Animations are a fundamental component of most games. The editor itself doesn’t have the ability to create animations, however they can be imported within fbx files. Once imported, an animation can only be used for the model it was imported on.

Sometimes it can be useful for an animation to trigger a specific behaviour in the game. For example, when a roll begins, it could trigger a period of invincibility for the player. One way of doing this is to embed a ‘begin roll’ and ‘end roll’ animation event into the clip. When replaying the animation, the animator will attempt to call the function associated with each event, which would toggle the invincibility of the player.

Audio Files

Audio files can be imported as either wav, mp3, ogg or flac files. The engine has a maximum limit of 8 audio files playing at once, after which files will be queued to be played when the next clip is finished. To use an audio file in script (the only way to use them), it must be loaded, read, and queued for play. If there are less than 8 audio clips playing, it will begin immediately, taking up a place in the buffer until the clip is complete.

Optionally, an audio clip can be set to play n amount of times, or an infinite amount of times. A fade-in and fade-out mechanic is coming soon.

Images

Image files can be imported as either png or jpg files, and have massive versatility in an engine. They can be used to texture models, giving higher resolution graphics without resorting to increasing the vertex count. They can be used as heightmaps for terrain, or as a skybox. Images can even be used totally arbitrarily, with a specific meaning given to each channel.

You don’t have to use all four channels in an image if you aren’t using them. During import you can select from RGBA, BGRA, RGB, RG or R.

Materials

Materials can either be imported from mtl files, or created in the engine. They can be shared between models, or if a model needs to be unique, it needs to have a unique material. A material describes how a mesh is rendered - things like diffuse, specular and shader model are all controlled by the material.
